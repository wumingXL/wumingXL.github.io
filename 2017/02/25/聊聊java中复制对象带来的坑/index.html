聊聊java中复制对象带来的坑

Feb 25, 2017 | Hits

#聊聊java中复制对象带来的坑

##背景：
  最近项目中有这么一个需求，我现在有两个表，一个定义表(define)，一个数据表(value)。定义表是从数据库中读取，数据表则从网络获取。定义表的作用是画UI，数据表的作用是存数据，但是，两个表对应的对象是一样的。

##操作：

  问题来了，当我新建一个界面的时候我是没有value的，那怎么办。灵机一动，既然是一个类直接把define复制过来不就行了，看我的。

  恩，复制好了，接下里可以进行各种操作了。(实际操作较复杂，这里简化)

  满心欢喜的敲完代码，一运行，我的乖乖，只看见屏幕刷的一下回到了上一个界面，很荣幸的程序崩溃了。怀着万分不解，我们去看看log，红色的NullPointException。

##反思：
java中数据类型可分为基本数据类型和引用数据类型。

  基本数据类型包含4类8种：

  整形：byte short int long

  浮点型：float double

  字符型：char

  bool型：boolean

  引用数据类型通常包含下面几种：

  类(class);

  接口(interface);

  数组(array);

基本数据类型传值的时候传递的是值得副本，引用数据类型传递的时候传递的是引用的副本。

所以在上述的情况中define和value指向同一个对象，修改对象的话，两个指针对应的值都改变。

##解决办法

  • (对javaBean进行操作)
    1.被复制的类需要实现Clonenable接口（不实现的话在调用clone方法会抛出CloneNotSupportedException异常) 该接口为标记接口(不含任何方法)
    2.覆盖clone()方法，访问修饰符设为public。方法中调用super.clone()方法得到需要的复制对象，（native为本地方法)
    3.如果对象里面包含其他引用类型则需要将其也进行1，2步操作。
  • （利用万能的gosn）
    1.将对象转换成json
    2.将json转换成对象

暂未对该两种方法对性能的影响做探究，本人用的是第二种方法，理由是简单，有什么更好的办法或者意见欢迎指出。
